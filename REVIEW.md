## Backend (NestJS)

### Структура

- [ ] Каждый модуль содержит: `*.module.ts`, `*.controller.ts`, `*.service.ts`.
- [ ] DTO находятся в `<module>/dto/`, декораторы в `<module>/decorators/`.
- [ ] Бизнес-сущности и ошибки домена — в `<module>/domain/`.
- [ ] CQRS: команды в `<module>/commands/`, запросы в `<module>/queries/`.

### API

- [ ] Все эндпоинты защищены `JwtAuthGuard`, кроме `/auth/login` и `/auth/register`.
- [ ] `@CurrentUser()` используется для получения пользователя из токена — не из body.
- [ ] DTO валидируются через `class-validator`; входные данные не доверяются без проверки.
- [ ] Контроллер не содержит бизнес-логики — только маршрутизация и вызов сервиса.

### База данных

- [ ] Все запросы к БД через `PrismaService` — не сырой SQL без необходимости.
- [ ] При удалении категории проверяется ограничение `Restrict` (есть транзакции → ошибка).
- [ ] Поле `amount` — `Decimal`, не `number`; при сериализации конвертируется в строку или число явно.
- [ ] Новые модели и изменения схемы сопровождаются миграцией (`prisma migrate dev`).

### Безопасность

- [ ] Нет утечки `passwordHash` в ответах API.
- [ ] Пользователь видит только свои данные: все запросы фильтруются по `userId` из токена.
- [ ] Нет SQL-инъекций (Prisma параметризует запросы — не конкатенировать строки вручную).

## Frontend (Next.js / React)

### Структура (FSD)

- [ ] Фичи находятся в `src/features/<name>/`, разбиты на `api/`, `model/`, `ui/`.
- [ ] Общие компоненты — в `src/shared/ui/`, утилиты — в `src/shared/lib/`.
- [ ] Страницы — в `src/app/(app)/` (защищённые) или `src/app/(auth)/` (публичные).
- [ ] Нет импортов из фичи в фичу — только через `shared` или пропсы.

### Данные и состояние

- [ ] Запросы к API через хуки в `features/<name>/api/` — не напрямую из компонентов.
- [ ] Токены не хранятся в `localStorage`; используется `httpOnly`-cookie или контекст сессии.
- [ ] Загрузочные и ошибочные состояния обработаны явно — нет "белого экрана" при ошибке.

### Компоненты

- [ ] UI-компоненты из `shadcn/ui` — не создавать свои аналоги примитивов (Button, Input, Dialog).
- [ ] Нет inline-стилей; стили через Tailwind-классы.
- [ ] Нет логики в JSX — вынести в переменные или функции выше `return`.
- [ ] Ключи в списках (`key`) — уникальные ID, не индексы массива.

### Типизация

- [ ] Пропсы компонентов явно типизированы — нет `React.FC` без generic или `props: any`.
- [ ] Типы ответов API берутся из `@expense-tracker/shared` — не дублировать локально.
