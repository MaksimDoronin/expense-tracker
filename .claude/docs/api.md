# API

Базовый URL: `http://localhost:3001` (настраивается через `NEXT_PUBLIC_API_URL`).

Swagger UI доступен по адресу `http://localhost:3001/api`.

Все защищённые маршруты требуют заголовка:
```
Authorization: Bearer <accessToken>
```

---

## Auth `/auth`

### POST /auth/register

Регистрация нового пользователя. Публичный.

**Body:**
```json
{
  "name": "Иван",
  "email": "ivan@example.com",
  "password": "secret123"
}
```

**Ответ 201:**
```json
{
  "user": { "id": "uuid", "name": "Иван", "email": "ivan@example.com" },
  "tokens": { "accessToken": "eyJ..." }
}
```

| Статус | Причина |
|--------|---------|
| 201    | Пользователь создан |
| 400    | Невалидные данные |
| 409    | Email уже занят |

---

### POST /auth/login

Вход в систему. Публичный.

**Body:**
```json
{
  "email": "ivan@example.com",
  "password": "secret123"
}
```

**Ответ 200:** та же структура, что у `/register`.

| Статус | Причина |
|--------|---------|
| 200    | Успешный вход |
| 400    | Невалидные данные |
| 401    | Неверный email или пароль |

---

### GET /auth/me

Данные текущего пользователя. **Требует JWT.**

**Ответ 200:**
```json
{
  "id": "uuid",
  "name": "Иван",
  "email": "ivan@example.com"
}
```

| Статус | Причина |
|--------|---------|
| 200    | OK |
| 401    | Токен отсутствует, истёк или пользователь не найден |

## Categories `/categories`

Все маршруты **требуют JWT**. Операции изолированы по пользователю.

### POST /categories

Создать категорию.

**Body:**
```json
{
  "name": "Продукты",
  "color": "#4ade80",
  "icon": "🛒"
}
```

**Ответ 201:**
```json
{
  "id": "uuid",
  "name": "Продукты",
  "color": "#4ade80",
  "icon": "🛒",
  "userId": "uuid",
  "createdAt": "2026-05-11T10:00:00.000Z",
  "updatedAt": "2026-05-11T10:00:00.000Z"
}
```

| Статус | Причина |
|--------|---------|
| 201    | Категория создана |
| 400    | Невалидные данные |
| 401    | Пользователь не аутентифицирован |
| 409    | Категория с таким именем уже существует |

---

### GET /categories

Список категорий текущего пользователя, отсортированных по `createdAt ASC`.

**Ответ 200:** массив объектов категории (структура — как при создании).

---

### PATCH /categories/:id

Частичное обновление категории.

**Body** (все поля опциональны):
```json
{
  "name": "Еда",
  "color": "#22c55e",
  "icon": "🍽️"
}
```

**Ответ 200:** обновлённая категория.

| Статус | Причина |
|--------|---------|
| 200    | Обновлено |
| 400    | Невалидные данные |
| 401    | Не аутентифицирован |
| 404    | Категория не найдена или не принадлежит пользователю |
| 409    | Имя уже занято |

---

### DELETE /categories/:id

Удалить категорию. Возвращает `204 No Content`.

> Удаление невозможно, если к категории привязаны транзакции (constraint `Restrict` в БД).

| Статус | Причина |
|--------|---------|
| 204    | Удалено |
| 401    | Не аутентифицирован |
| 404    | Не найдено |
| 409    | Удаление невозможно — к категории привязаны транзакции |

## Transactions `/transactions`

Все маршруты **требуют JWT**. Операции изолированы по пользователю.

### POST /transactions

Создать транзакцию.

**Body:**
```json
{
  "amount": 1500.50,
  "type": "expense",
  "date": "2026-05-11",
  "categoryId": "uuid",
  "description": "Продукты в магазине"
}
```

Поля:
- `amount` — положительное число, до 2 знаков после запятой
- `type` — `"income"` | `"expense"`
- `date` — ISO 8601 (строка)
- `categoryId` — UUID категории, принадлежащей текущему пользователю
- `description` — опционально, до 500 символов

**Ответ 201:**
```json
{
  "id": "uuid",
  "amount": "1500.50",
  "type": "expense",
  "description": "Продукты в магазине",
  "date": "2026-05-11T00:00:00.000Z",
  "categoryId": "uuid",
  "userId": "uuid",
  "createdAt": "2026-05-11T10:00:00.000Z"
}
```

> `amount` возвращается строкой для точной передачи Decimal.

| Статус | Причина |
|--------|---------|
| 201    | Создано |
| 400    | Невалидные данные |
| 401    | Не аутентифицирован |
| 404    | Категория не найдена или не принадлежит пользователю |

---

### GET /transactions

Список транзакций с фильтрацией, отсортированных по `date DESC`.

**Query-параметры** (все опциональны):
| Параметр     | Тип    | Описание |
|--------------|--------|----------|
| `dateFrom`   | string | Нижняя граница дат (ISO 8601, включительно) |
| `dateTo`     | string | Верхняя граница дат (ISO 8601, включительно) |
| `type`       | string | `"income"` или `"expense"` |
| `categoryId` | string | UUID категории |

Фильтры применяются по AND. Без параметров — все транзакции пользователя.

**Ответ 200:** массив транзакций (структура — как при создании).

---

### GET /transactions/summary

Агрегированная сводка за календарный месяц.

**Query-параметры (обязательные):**
| Параметр | Тип    | Описание |
|----------|--------|----------|
| `month`  | number | 1–12 |
| `year`   | number | 1970–2100 |

**Ответ 200:**
```json
{
  "month": 5,
  "year": 2026,
  "totalIncome": "50000.00",
  "totalExpense": "23500.75",
  "balance": "26499.25",
  "byCategory": [
    { "categoryId": "uuid", "type": "expense", "total": "5000.00" }
  ]
}
```

Все суммы — строки с двумя знаками после запятой. `balance` может быть отрицательным.

| Статус | Причина |
|--------|---------|
| 200    | OK |
| 400    | Невалидные параметры |
| 401    | Не аутентифицирован |

---

### GET /transactions/:id

Получить транзакцию по UUID.

**Ответ 200:** объект транзакции.

| Статус | Причина |
|--------|---------|
| 200    | OK |
| 401    | Не аутентифицирован |
| 404    | Не найдена или не принадлежит пользователю |

---

### PATCH /transactions/:id

Частичное обновление транзакции. Тело — те же поля, что при создании, все опциональны.

**Ответ 200:** обновлённая транзакция.

| Статус | Причина |
|--------|---------|
| 200    | Обновлено |
| 400    | Невалидные данные |
| 401    | Не аутентифицирован |
| 404    | Транзакция или категория не найдены |

---

### DELETE /transactions/:id

Удалить транзакцию. Возвращает `204 No Content`.

| Статус | Причина |
|--------|---------|
| 204    | Удалено |
| 401    | Не аутентифицирован |
| 404    | Не найдена или не принадлежит пользователю |
