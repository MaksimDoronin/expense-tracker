Создай git-коммит для текущих изменений.

## Алгоритм выполнения

### Контекст выполнения

Статус проекта: !`git status`
Последние коммиты !`git log --oneline -10`

### Шаги

1. Запусти `git diff` - разберись в сути изменений
2. Определи `type` и `scope` по таблицам ниже.
3. Сформулируй `description` на русском, в нижнем регистре, без точки в конце.
4. Добавь нужные файлы в индекс (`git add <files>`), не используй `git add -A` или `git add .`.
5. Создай коммит через HEREDOC:

```bash
git commit -m "$(cat <<'EOF'
<type>(<scope>): <description>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

7. Проверь результат через `git status`.

## Формат

```
<type>(<scope>): <description>
```

## Типы

| Тип        | Когда использовать                              |
|------------|-------------------------------------------------|
| `feat`     | Новая функциональность                          |
| `fix`      | Исправление бага                                |
| `refactor` | Рефакторинг без изменения поведения             |
| `chore`    | Настройка инструментов, зависимости, конфиги    |
| `docs`     | Только документация                             |
| `test`     | Добавление или исправление тестов               |
| `perf`     | Улучшение производительности                    |
| `style`    | Форматирование, пробелы (не влияет на логику)   |

## Scope

Имя затронутого модуля: `backend`, `frontend`, `shared`, `prisma`, `auth`, `categories`, `transactions` и т.д. Можно опустить, если изменения глобальные.

## Правила

- `description` — на **русском языке**, нижний регистр, без точки в конце
- Один коммит = одна логическая единица изменений
- Breaking changes: `!` после типа — `feat(api)!: rename endpoint`

## Запрещено
- **Никогда не коммить** - `.env`, `.env.local`, файлы с секретами
- **Никогда не пушить** автоматически
- **Не использовать** --no-verify или --amend без явной просьбы
- **Не добавлять** файлы без понимания их содержимого
