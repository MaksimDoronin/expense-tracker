Создай Pull Request на GitHub.

## Аргументы скилла

Аргументы передаются после `/pr` через пробел:
- $0 — название PR (title).
- $1 — имя целевой ветки.

## Подготовка

1. проверь что ветка готова:
`bash ${CLAUDE_SKILL_DIR}/scripts/validate.sh` 

## Шаги

1. Проверить, что текущая ветка не main
2. Убедиться, что ветка запушена в remote
```bash
git status -sb
```

- если нет upstream - запушить

3. Получить список коммитов до HEAD `git log main...HEAD --oneline`
4. Получить полный diff
5. На основе коммитов составить описание PR:
- **итог** - 2-4 пункта, что реализовано/исправлено (суть, а не файлы)
- **Изменения** - затронутые модули, Endpoint, (если есть)
- **План тестирования** - Краткий чек-лист шагов для проверки. 
6. Создать PR: gh pr create --title "$0" --base "$1" --body "..."
7. Вывести URL созданного PR

Если $1 не передан — использовать main как целевую ветку.
Если PR для этой ветки уже существует — сообщить и вывести ссылку.

## Правила

- Title если не передан то в формате Conventional Commits: `<type>(<scope>): описание на русском`.
- Никогда не пушить в `main` напрямую.
- После создания обязательно вернуть URL PR пользователю.
- Если PR для этой ветки уже существует — сообщить пользователю и показать ссылку через `gh pr view --web` или `gh pr view <branch>`.
